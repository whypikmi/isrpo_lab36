### Описание проекта
**TaskBoard** — это интерактивная доска задач (аналог Trello в минималистичном исполнении), которая позволяет пользователю:
- **Просматривать** актуальный список задач с сортировкой (невыполненные — сверху, самые свежие — первыми)
- **Добавлять** новые задачи с заголовком и описанием
- **Отмечать** задачи как выполненные (визуальное перечёркивание)
- **Редактировать** текст задачи прямо в интерфейсе (с возможностью отмены)
- **Удалять** задачи с подтверждением
- **Фильтровать** задачи: все / в работе / выполненные
- **Видеть счётчик** выполненных задач

Проект построен по **Fullstack-архитектуре**:
- **Backend:** ASP.NET Core Web API + Entity Framework Core + SQLite
- **Frontend:** HTML + CSS (Grid Layout) + JavaScript (Fetch API)
### Инструкция по запуску

```

# Восстановите зависимости (если не сделано ранее)
```

**Вариант 1:**  
Используйте расширение **Live Server** в VS Code — откройте `frontend/index.html` и нажмите "Go Live".

Вручную откройте файл `frontend/index.html` в браузере (убедитесь, что CORS настроен корректно — в `Program.cs` уже добавлена политика `AllowAll`).

### Главные выводы
1. **Entity Framework Core значительно упрощает работу с базой данных.**  
Вместо написания SQL-запросов вручную мы описываем модели на C#, создаём миграции одной командой, а всё остальное EF Core делает сам — от создания таблиц до выполнения запросов через LINQ.
2. **Асинхронное программирование (async/await) критически важно для отзывчивого UI.**  
Когда сервер выполняет `await _db.Tasks.ToListAsync()`, он не блокирует поток, а может обрабатывать другие запросы. На фронтенде `await fetch(...)` не "замораживает" страницу — пользователь может взаимодействовать с интерфейсом пока идёт загрузка.
3. **Разделение ответственности через модульную архитектуру упрощает поддержку кода.**  
В проекте выделены отдельные модули: `api.js` отвечает только за сетевые запросы, `app.js` — за логику интерфейса и состояние, CSS — за внешний вид. Это делает код понятным и тестируемым.
4. **CORS (Cross-Origin Resource Sharing) — это не баг, а особенность безопасности браузеров.**  
Когда фронтенд (например, `http://127.0.0.1:5500`) пытается обратиться к API на `http://localhost:5000`, браузер блокирует запрос. Это защита. Без явного разрешения через `AddCors()` в бэкенде приложение не заработает.
5. **Хранение состояния на фронтенде (allTasks + activeFilter) позволяет фильтровать без лишних запросов.**  
Вместо того чтобы при каждом клике на фильтр бежать на сервер за новыми данными, мы загружаем все задачи один раз, а затем просто изменяем отображаемый массив через `filter()`. Это быстрее и снижает нагрузку на сервер.
6. **Два режима отображения одной карточки (просмотр/редактирование) — удобный UX-паттерн.**  
Чередование блоков через `display: flex` / `display: none` с сохранением обоих DOM-элементов позволяет переключаться без пересоздания HTML, сохраняя введённые данные при ошибке валидации.

7. **Git с частыми коммитами — это страховка и история развития проекта.**  
   Выполняя `git add`, `git commit` и `git push` после каждого шага, мы получаем возможность в любой момент откатиться к рабочей версии, а также видим эволюцию кода — от пустого шаблона до полноценного приложения.
